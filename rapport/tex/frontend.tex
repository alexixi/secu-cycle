\section{Frontend}
\label{sec:frontend}
\noindent Dans un premier temps, nous avons développé un site Web, présenté dans la Section~\ref{sec:web}.
Une fois que la première version du site était prête, nous avons commencé le développement 
d'une application mobile, présentée dans la Section~\ref{sec:mobile},
facilitant l'utilisation sur smartphone.

\subsection{Site Web}
\label{sec:web}
\noindent L'application Web a été développée en utilisant React et Vite.\\

\noindent Le site Web est composé d'une page d'accueil présentant le projet.
Il y a aussi une page de connexion, nécessitant une adresse mail et un mot de passe.
Si l'utilisateur n'a pas encore de compte, il peut en créer un via la page de création de compte.
Il y a, de plus, une page consacrée à la recherche d'itinéraires (Section~\ref{sec:webrecherche}) et une page de profil utilisateur (Section~\ref{sec:webprofil}).

\subsubsection{Page de recherche d'itinéraires}
\label{sec:webrecherche}
\noindent La page principale de l'application est celle permettant la recherche d'itinéraire (Figure~\ref{fig:webrecherche}).

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{img/PageItineraire.png}
    \caption{Capture d'écran de la page de recherche d'itinéraire.}
    \label{fig:webrecherche}
\end{figure}

\noindent La partie gauche de cette page contient le menu de recherche, permettant d'entrer les adresses de départ et d'arrivée du trajet, l'heure maximale d'arrivée ou la durée maximale de trajet, ainsi que le type de vélo utilisé.
Les adresses sont autocomplétées grâce à l'utilisation de l'API Base Adresse Nationale (BAN \cite{BAN}), proposée par le gouvernement français.
En appuyant sur le bouton "Calculer les itinéraires", l'utilisateur envoie la requête de recherche d'itinéraires à l'API.
Les trajets proposés sont ensuite affichés sur la carte, et leur description apparaît dans la partie gauche de l'écran.\\

\noindent Un trajet est décrit par son type : rapide, sécurisé ou un compromis entre les deux.
Nous affichons la dénivelée positive et négative rencontrée lors du trajet, la distance et le temps de parcours estimé.
Celui-ci dépend du profil sportif de l'utilisateur.
Nous affichons aussi le pourcentage de pistes cyclables composant le trajet, le pourcentage de zone où la vitesse des véhicules est limitée à 30 km/h ainsi que le pourcentage de voies éclairées.\\

\noindent De plus, l'utilisateur a accès à un petit graphique représentant l'évolution de la dénivelée pendant le trajet.
En cliquant sur le bouton d'information situé à côté du type de trajet, un texte récapitulant les caractéristiques de sécurité du trajet apparaît.\\

\noindent De plus, la page de recherche d'itinéraire propose une carte.
Sur celle-ci s'affichent les différents trajets suggérés.
Un cours descriptif du trajet apparaît quand la souris le survole.
Un clique sur le trajet provoque la sélection du trajet et l'apparition du descriptif détaillé du trajet dans la partie gauche de l'écran.\\

\noindent Sur la carte se trouvent trois boutons principaux.

\noindent Le bouton "Trafic en temps réel" déclenche l'apparition des points de trafic ainsi que de leur légende.
Les données du trafic sont récupérées grâce à l'API Avatar \cite{Avatar} proposée par le Cerema.
Ces données sont utilisées avec une heure et demie de décalage et sont actualisées toutes les cinq minutes.\\

\noindent On trouve aussi sur la carte le bouton "Ajouter un signalement".
Une fois ce bouton cliqué, l'utilisateur peut cliquer à un endroit sur la carte.
Il peut alors créer un signalement, visible par les autres utilisateurs et pris en compte dans le calcul des itinéraires.
L'utilisateur peut sélectionner le type de signalement parmi "Accident", "Travaux", "Danger" et "Obstacle", et entrer s'il le souhaite une description de l'évènement.\\

\noindent Le bouton "Calques" permet de modifier le fond de carte.
L'utilisateur peut donc l'ajuster selon ses goûts.
Il peut choisir entre une carte sombre, une carte avec le relief, les rues, la vue satellite...

\subsubsection{Page profil utilisateur}
\label{sec:webprofil}
\noindent La seconde page du site Web est la page présentant le profil de l'utilisateur (Figure~\ref{fig:webprofil}).

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{img/PageUtilisateur.png}
    \caption{Capture d'écran de la page de profil de l'utilisateur.}
    \label{fig:webprofil}
\end{figure}

\noindent À partir de cette page, l'utilisateur peut accéder à ses informations.
Un clic sur le bouton "Modifier mon compte" provoque l'ouverture d'une fenêtre modale permettant à l'utilisateur de modifier son nom, son adresse mail, sa date de naissance, son mot de passe et son niveau sportif (Débutant, Intermédiaire ou Avancé).

\noindent L'utilisateur a aussi accès à ses adresses de préférence, qui lui sont recommandées lors de la recherche d'itinéraire pour permettre une utilisation quotidienne rapide.
Il peut modifier ses adresses via le bouton "Modifier mes adresses".

\noindent L'utilisateur peut aussi enregistrer ses vélos, attribuer à chacun un nom personnalisé, un type (Ville, VTT ou Route) et indiquer s'il est ou non électrique.
Les vélos de l'utilisateur lui sont proposés lors de la recherche d'itinéraire.\\

\noindent Sur la page de profil se trouve l'historique de l'utilisateur, ainsi que ses statistiques, directement calculées par rapport à l'historique.
L'utilisateur a aussi le choix de modifier son historique.
Lors d'un clic sur un trajet de celui-ci, une fenêtre modale s'ouvre, lui permettant d'accéder à toutes les informations du trajet effectué (Figure~\ref{fig:historic}).
